Barycentric Search Engine

This page is a compact search interface for the barycentric coordinates already present in the repository viewer logic. It follows the notebook's idea of treating a center as a searchable entity with a coordinate triple, then filtering by center ID, center name, or the coordinate text itself.

Search logic

The dataset is stored as colon-separated triples such as \(a:b:c\) or \(b+c-a:c+a-b:a+b-c\). That mirrors the notebook/pipeline convention where barycentrics are normalized into a searchable text form before further symbolic conversion.

Examples

  • Search X(8) to jump straight to the Nagel point.
  • Search b+c-a to find Nagel/Mittenpunkt-style expressions.
  • Use the prefix filter to restrict results to a family like X(1.
