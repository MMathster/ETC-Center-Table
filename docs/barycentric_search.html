Barycentric Search Engine Notebook-style extraction, page-aware filtering, graceful fallback

This page now tries to parse the live ETC pages with the same broad strategy as the Python notebook: walk the coordinate block, isolate genuine Barycentrics lines, reject prose/cross-references, and keep every barycentric variant found for a center. When the browser cannot reach those pages, it falls back to the generated local index bundled with this repo.

What changed

The search dataset is no longer treated as a flat hand-entered table. Each result now keeps a source page, all extracted barycentric variants, and combined search text for notebook-like filtering.

How to use it

  • Filter by X(8) or any center name.
  • Limit the search to one ETC page such as ETC.html.
  • Switch between center-name, page, and barycentric-only queries.

Why there is a fallback

Some browsers or environments cannot fetch the upstream ETC HTML directly. The local JSON keeps the search page usable while preserving the same UI and filter model.

Loading pipeline

The interface below shows whether the page is using live ETC HTML parsing or the local fallback index. The progress bar and section highlighting update while the loader switches modes.

Notebook-inspired flow
1. Fetch ETC.html, ETCPart2.html, …
2. Walk each center header until the next structural block.
3. Keep only direct barycentric triples and expand search text from every variant.
Section effects

The loader and results sections fade and glow into focus as they become active, so switching between the search states feels less abrupt.

Page-aware search

The page filter is populated from the loaded dataset, so the UI reflects whatever pages were actually indexed.
